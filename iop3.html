Types of Errors in C

1. Compilation Errors

Compilation errors occur during the compilation stage, preventing the code from being converted into machine language.

            
#include <stdio.h>

int main() {
    printf("Hello, World!") // Missing semicolon
    return 0;
}

Why is this an error? The compiler expects a semicolon (`;`) at the end of the `printf` statement. Without it, the code won't compile.

2. Syntax Errors

Syntax errors arise when the C language rules are violated, such as missing brackets or incorrect keywords.

            
#include <stdio.h>

int main() {
    int 1number = 10; // Variable names cannot start with a number
    printf("%d", 1number);
    return 0;
}

Why is this an error? Variable names cannot start with a number, causing a syntax error.

3. Linking Errors

Linking errors occur when the compiler cannot find the definition of a function or variable used in the program.

            
#include <stdio.h>

int main() {
    myFunction(); // No definition provided
    return 0;
}

Why is this an error? The function `myFunction()` is called but not defined, leading to a linker error.

4. Runtime Errors

Runtime errors occur while the program is running, such as division by zero or accessing invalid memory locations.

            
#include <stdio.h>

int main() {
    int a = 5, b = 0;
    printf("%d", a / b); // Division by zero
    return 0;
}

Why is this an error? Division by zero is undefined and causes the program to crash.

5. Other Common Errors

  • Segmentation Fault: Accessing memory that doesn't belong to the program.
  • Uninitialized Variables: Using a variable without assigning a value.
  • Infinite Loops: A loop without a proper termination condition.
  • Type Mismatch: Assigning a float value to an int variable without typecasting.

Recursion in C

Recursion is a process in which a function calls itself until a base condition is met. It is commonly used for tasks like factorial, Fibonacci series, and tree traversal.

Factorial Using Recursion

            
#include <stdio.h>

int factorial(int n) {
    if (n == 0) return 1; // Base case
    return n * factorial(n - 1);
}

int main() {
    int num = 5;
    printf("Factorial of %d is %d", num, factorial(num));
    return 0;
}

Factorial Without Recursion

            
#include <stdio.h>

int main() {
    int num = 5, fact = 1;
    for (int i = 1; i <= num; i++) {
        fact *= i;
    }
    printf("Factorial of %d is %d", num, fact);
    return 0;
}

Fibonacci Series Using Recursion

            
#include <stdio.h>

int fibonacci(int n) {
    if (n == 0) return 0;
    if (n == 1) return 1;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

int main() {
    int n = 10;
    printf("Fibonacci Series: ");
    for (int i = 0; i < n; i++) {
        printf("%d ", fibonacci(i));
    }
    return 0;
}

How it works? The function calls itself for `n-1` and `n-2` until it reaches base cases (`0` and `1`). The results are added to generate the series.

Sum of Digits Using Recursion

            
#include <stdio.h>

int sumOfDigits(int n) {
    if (n == 0) return 0; // Base case
    return (n % 10) + sumOfDigits(n / 10);
}

int main() {
    int num = 1234;
    printf("Sum of digits: %d", sumOfDigits(num));
    return 0;
}

Sum of Digits Without Recursion

            
#include <stdio.h>

int main() {
    int num = 1234, sum = 0;
    while (num > 0) {
        sum += num % 10; // Add last digit
        num /= 10;       // Remove last digit
    }
    printf("Sum of digits: %d", sum);
    return 0;
}

How it works? The loop extracts the last digit using `%10`, adds it to `sum`, and then removes the last digit using `/10` until `num` becomes `0`.
